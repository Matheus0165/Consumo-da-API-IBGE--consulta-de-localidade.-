Consulta de Localidades
Brasileiras

Projeto que consulta em tempo real a base de dados oficial do IBGE para listar os 26 estados, o Distrito Federal e todos os municípios brasileiros — sem chave, sem autenticação, apenas requisições HTTP GET com respostas em JSON.

Fonte IBGE — v1
Método HTTP GET
Formato JSON · UTF-8
Autenticação Não requerida

Análise da API

Características técnicas da API de Localidades do IBGE

01

Propósito

Disponibilizar dados oficiais sobre a divisão político-administrativa do Brasil — regiões, estados, mesorregiões, microrregiões, municípios, distritos e subdistritos.

02

Base URL

https://servicodados.ibge.gov.br/api/v1/localidades

03

Principais endpoints

  • GET /estados — lista todos os estados e o DF
  • GET /estados/{UF} — detalhes de um estado pela sigla
  • GET /estados/{UF}/municipios — municípios de um estado
  • GET /regioes — lista as 5 regiões do Brasil
  • GET /municipios — lista todos os municípios brasileiros
  • GET /estados/{UF}/distritos — distritos de um estado
04

Possui autenticação?

Não. A API é pública e gratuita, sem necessidade de cadastro ou credenciais para consumo.

05

Tipo de autenticação

Nenhum. Não exige API Key, Bearer Token, OAuth, Basic Auth ou qualquer outro mecanismo.

06

Onde a credencial é enviada

Não se aplica. Como não há autenticação, nenhum dado é enviado em header, query string ou body.

07

Versionamento

Sim — via URI (URI Versioning). A versão é declarada no próprio caminho da URL: /api/v1/. Mudanças incompatíveis criariam um /v2/ em paralelo, mantendo a versão anterior funcionando.

08

Utiliza HATEOAS?

Não. A resposta retorna apenas dados, sem links de hipermídia (_links, href) apontando para recursos relacionados. O cliente precisa montar manualmente a URL para navegar entre os recursos.

09

Tipo de resposta

JSON com codificação UTF-8. Header retornado: Content-Type: application/json; charset=utf-8.
